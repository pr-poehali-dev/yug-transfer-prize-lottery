# Пакет для переноса раздела «Посты в канал» в новый проект

Этот документ — пошаговая инструкция и список всего, что нужно перенести,
чтобы раздел «Посты в канал» (посты, истории, исключённые водители,
ежедневный автопостинг, публикация в VK) заработал в новом проекте poehali.dev.

> ВАЖНО: перенос между проектами делается вручную. Создай новый проект,
> затем в нём попроси разработчика подключить всё по этому списку.

---

## 1. Порядок действий (коротко)

1. Создай новый пустой проект на poehali.dev.
2. Создай в нём таблицы БД — выполни SQL из раздела 4 (файл `db_migrations/V0001__posts_module.sql` уже готов в этом пакете).
3. Добавь секреты из раздела 3 (значения возьми из текущего проекта — Ядро → Секреты).
4. Перенеси backend-функции из раздела 2 (скопировать папки из `/backend`).
5. Перенеси frontend-файлы из раздела 5.
6. Подключи страницу `/posts` и (по желанию) свой домен.

---

## 2. Backend-функции (папки из /backend)

Скопировать целиком вместе с `index.py`, `requirements.txt`, `tests.json`:

| Папка | Назначение |
|-------|-----------|
| `admin-auth` | Вход в раздел (scope=posts: POSTS_LOGIN/POSTS_PASSWORD) |
| `admin-posts` | Посты в Telegram-канал (CRUD, публикация, загрузка фото, расписание) |
| `admin-bot-posts` | Ежедневные посты бота @ug_sait_bot (CRUD) |
| `sait-bot-daily` | Автопубликация ежедневных постов в Telegram + ВКонтакте |
| `admin-bot-stories` | Управление видео-сторис |
| `stories-cron` | Автопубликация сторис раз в 48 часов (Telethon) |
| `excluded-watcher` | Авто-сообщения исключённым водителям (Telethon listener) |
| `tg-user-auth` | Логин Telegram user-аккаунта №1 (для сторис) |
| `tg-user-auth2` | Логин Telegram user-аккаунта №2 (для исключённых) |
| `tg-accounts` | Пул user-аккаунтов Telegram |
| `upload-video` | Загрузка видео-кружка в S3 |

После копирования вызвать деплой бэкенда — платформа выдаст новые URL.
Эти новые URL нужно вписать в `src/components/admin/adminTypes.ts` (см. раздел 5).

## 3. Секреты (Ядро → Секреты в новом проекте)

Значения скопировать из текущего проекта.

| Секрет | Для чего |
|--------|----------|
| `DATABASE_URL` | подключение к БД (создаётся платформой автоматически) |
| `MAIN_DB_SCHEMA` | имя схемы БД (создаётся платформой автоматически) |
| `AWS_ACCESS_KEY_ID` | хранилище S3 (создаётся платформой автоматически) |
| `AWS_SECRET_ACCESS_KEY` | хранилище S3 (создаётся платформой автоматически) |
| `ADMIN_LOGIN` | логин администратора |
| `ADMIN_PASSWORD` | пароль администратора |
| `POSTS_LOGIN` | логин раздела «Посты» (Yaltataran) |
| `POSTS_PASSWORD` | пароль раздела «Посты» (Taran220577) |
| `TELEGRAM_BOT_TOKEN` | токен главного бота |
| `UG_INFO_BOT_TOKEN_NEW` | токен бота @ug_info_bot для постов в канал |
| `TELEGRAM_BOT_TOKEN_2` | токен бота @ug_sait_bot (сайт/сторис/ежедневные) |
| `UG_DRIVER_CHANNEL_ID` | канал для постов (@UG_DRIVER) |
| `TELEGRAM_CHANNEL_ID` | канал для розыгрышей/сторис |
| `TELEGRAM_BUSINESS_CONNECTION_ID` | публикация сторис через business-аккаунт |
| `TG_API_ID` | Telegram API ID (Telethon) |
| `TG_API_HASH` | Telegram API Hash (Telethon) |
| `VK_USER_TOKEN` | токен ВКонтакте |
| `VK_GROUP_ID` | ID сообщества ВК |
| `VK_USER_ID` | ID пользователя ВК |

> Первые 4 секрета (DATABASE_URL, MAIN_DB_SCHEMA, AWS_*) платформа создаёт сама
> при создании проекта — их вписывать вручную не нужно.

## 4. База данных

Готовый файл миграции лежит в этом пакете: `db_migrations/V0001__posts_module.sql`.
Выполни его в новом проекте через инструмент миграций.

Таблицы, которые он создаёт:
`posts`, `bot_daily_posts`, `bot_stories`, `excluded_drivers`, `excluded_settings`,
`tg_user_session`, `tg_user_session2`, `tg_user_session3`, `tg_user_accounts`,
`business_connections`, `app_settings`.

> Данные (сами посты, истории, список исключённых) — это содержимое таблиц.
> Если нужно перенести и их, экспортируй строки SELECT-ом из старого проекта
> и вставь INSERT-ом в новый. Сессии Telegram (`session_string`) лучше
> залогинить заново в новом проекте, а не копировать.
